# Contributing to Project Labyrinth Sentinel

Thank you for your interest in contributing to Project Labyrinth Sentinel (PLS). As an open-source framework for cognitive defense, we value contributions that enhance the resilience and integrity of the system.

## How to Contribute

### 1. Reporting Bugs
If you find a bug, please open an issue on GitHub. Include:
- A clear description of the issue.
- Steps to reproduce the bug.
- Any relevant logs or screenshots.

### 2. Suggesting Enhancements
We welcome ideas for new features or protocol improvements (e.g., HLSP optimizations). Please open an issue to discuss your proposal before starting implementation.

### 3. Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or fix.
3. Ensure your code follows **PEP 8** and includes proper **typing** and **logging**.
4. Submit a pull request with a detailed description of your changes.

## Development Guidelines
- **Security First**: Ensure no sensitive data is leaked in logs. Strictly avoid using `eval()` or similar dangerous functions.
- **Documentation**: Update the README or relevant documentation for any changes.
- **Testing**: Include tests for any new logic, especially within the Vector Profiler or HLSP modules.

## 2030 Compliance
All contributions must be compatible with the **2030 Long-term Execution Directive**. Ensure that administrative sanitization and lineage tracking are maintained in your code.
